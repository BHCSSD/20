## Step 1: Grid Layout Base

> [!NOTE]
> ### What is a Nested Array?
> 
> A **nested array** (aka a multi-dimensional array) is **an array inside of another array**. Instead of holding a list of simple data types like numbers or strings, the parent array holds other arrays as its elements.
> 
> 
> ### How to Visualize It
> 
> ```javascript
> let grid = [
>    ["Row 0, Col 0", "Row 0, Col 1"],
>    ["Row 1, Col 0", "Row 1, Col 1"],
>    ["Row 2, Col 0", "Row 2, Col 1"]
> ]
> 
> ```
> 
> ### How to Access the Data (Bracket Notation)
> 
> To get data out of a nested array, you use **two sets of square brackets** `[row][column]`:
> 1. The **first** bracket selects the inner array (the row).
> 2. The **second** bracket selects the element inside that inner array (the column).
> 
> ### Why Use Them?
> 
> Nested arrays are perfect for showing grids, board games, pixel data, or coordinate pairs where pieces of information belong grouped together.

## Step 5: Grid-Aware Collision Helper

> [!NOTE]
> ### What is AABB Collision Detection?
> 
> 
> **AABB** aka **Axis-Aligned Bounding Box**. It is the simplest math formula used to check if two rectangles are overlapping.
> It works by looking at the four edges of the rectangles. A collision is happening **unless** one box is completely to the left, right, above, or below the other box.
> ### Optimization: Distance Skipping
> 
> Running bounding box math against thousands of objects on a massive map can cause severe performance lag. To fix this, we calculate the rough distance between the player and an obstacle first. If the obstacle is further than 2 tiles away, we skip (`continue`) the complex overlap math entirely.

* This helper checks a single layout category array for collisions.
* Because the array contains grid coordinates `[col, row]`, it multiplies them by `size` (`cellSize`) inside the loop to convert them to pixel coordinates before evaluating bounding-box properties.

```javascript
function checkCollisions(px, py, gridArray, size) {
  for (let i=0; i<gridArray.length; i++) {
    // Map grid positions back to absolute engine pixels
    let obstacleX
    let obstacleY

    // OPTIMIZATION: Check the absolute distance between player and block
    let distanceX
    let distanceY

    // If the object is too far away, skip evaluating it!

    }

    //AABB collision math

        py < obstacleY + size &&

      return true // Collision detected
    }//end if
  }//end for
  return false // No collision found
}//end checkCollisions
```
