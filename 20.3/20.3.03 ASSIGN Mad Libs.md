```
20.3.03 ASSIGN Mad Libs
```

You will create a **Mad Libs app** that:
1. Lets a user upload a `.txt` story that contains **smart words** like `<noun>`, `<verb>`, `<adjective>`.
2. Displays the uploaded story on screen.
3. Replaces those smart words with the user’s custom inputs.
    * Make sure you have at least **6 blanks** to fill 
4. Displays the **new, completed story** on screen.

---

## Step 1: Example Mad Libs Template

Create a text file `madlib.txt` with placeholder “smart words.”  
These should be surrounded by angle brackets `< >`. 
* Why you may ask? Easy standardization for me.
* **Example `madlib.txt`:**
   ```text
   Once upon a time, there was a <noun> who loved to <verb> every single day.
   One morning, the <noun> found a <adjective> treasure in the backyard!

   ```

* Add an upload button to load your `madlib.txt`.

## Step 3: Replace the Smart Words

Make a `goMad` button that pulls all the user inputs and replaces the smart words.

* Make sure to replace *all* occurrences of the smart word.

> [!NOTE]
> ### Coding with Efficiency: `return` vs. `continue`
> 
> When processing text or inputs, we want our code to be smart. Instead of running heavy string manipulation code unnecessarily, we can use control keywords to break out early and save processing power.
> #### 1. The "Early Return" Pattern (For Functions)
> 
> If a textbox is empty, we can use `return` to escape a function immediately and skip the rest of its code.
> ```javascript
> function processWord(placeholder, userInput) {
>   // IF the user didn't type anything, skip the rest of the function!
>   if (userInput=== "") {
>     return // Stops execution right here
>   }
>   
>   // Otherwise, proceed with the replacement
> }
> 
> ```
> 
> *Using `return` like this prevents bugs and stops your app from replacing placeholders with blank spaces.*
> 
> 
> #### 2. The "Continue" Pattern (For Loops)
> 
> If you are looping through an array of multiple placeholders, you can't use `return` (because that would kill the whole function). Instead, use `continue` to **skip the current item** and jump straight to the next turn of the loop.
> ```javascript
> let targets = ["<noun>", "<verb>", "<adjective>"];
> 
> for (let i=0; i<targets.length; i++) {
>   //IF the story doesn't even contain this tag, skip it
>   if ( !storyText.includes(targets[i]) ) {
>     continue// Jumps directly to the next 'i', skipping the code below
>   }
>   
>    dooStuff()// This code only runs if the target was actually found
> }
> 
> ```
