## **Step 1: Data Integration**
* **The Dataset:** Use `loadJSON()` to import the full National Pokédex
* **Async Loading:** Ensure your program uses `async` and `await` when loading images to prevent the UI from freezing or crashing.
* **Loading State:** Implement an if/else statement to verify the data has successfully loaded before attempting to display it.

## **Step 2: Interactive Controls**
Build a control panel with buttons to perform the following:
* **Sequential Navigation:** "Next" and "Previous" buttons to cycle through IDs.
* **The Chaos Factor:** A "Random" button to jump to any Pokémon in the list.
* **Localization:** A button to toggle the Pokémon’s name between **English, Japanese, Chinese,** and **French**.
* **Filtered Random:** for the higher level- A dropdown menu (`createSelect`) where a user picks a type like "Fire".
  * When the Random action is triggered with a filter active, the app must pick a random Pokémon *only* from that specific type.

> [!TIP]
> ### **How to populate your pulldown `createSelect` aka The Grocery Store Method**
> Imagine you are walking through a store. You have a list, and you only want to write down **new** items you see.
> 1. **Pick up a Pokémon.**
> 2. **Look at its type** like "Fire"
> 3. **Check your list:** "Is 'Fire' already on my list?"
> 4. **Action:** If the answer is **No**, write it down. If the answer is **Yes**, ignore it and move to the next Pokémon.
> 
> * **Implementation:** Use a standard `for` loop and an empty array to store unique types as you find them.
> * **Optimization:** You must **ONLY DO THIS ONCE** (ideally right after the JSON loads) otherwise you will impact performance.
> * **Logic:** The `.includes()` method and the `!` **NOT** operator are your best friends here.
>


## **Step 3: Adaptive UI & Data Display**
Your interface should feel like a high-tech encyclopedia:
* **Visuals:** Display the `hires` image.
* **Bio-Data:** Show the Species description, Height, and Weight.
* **Description** add in a text block
* **Dynamic Theming:** The background or UI accents must change color based on the Pokémon’s primary type
  *  something like Green for Grass, Blue for Water).
* **Abilities:** If the Pokémon has a special `ability`, display it clearly on the screen.


## **Step 4: Global Type Analytics**
To achieve full marks, your Pokédex must analyze the entire database in real-time:
* **The Analysis Function:** Create a function that scans the *entire* Pokédex to calculate the **average** HP, Attack, and Defense for all Pokémon sharing the current Pokémon's primary type.
* **The Comparison:** Display the current Pokémon's stats side-by-side with the **Global Type Average**.
* **Visual Feedback:** Clearly indicate (using text or icons) if your Pokémon is **"Above Average"** or **"Below Average"** compared to its peers.
* **Efficiency:** Calculations should only run when the Pokémon changes **DO NOT** recalculate 60 times a second in the `draw` loop


## Starter
```js
let pokeDex;
let pokeImg;
let currentIndex = 0; //which poke to pick


async function loadStuff() {
  pokeDex = await loadJSON("pokedex.json");
  // Load the initial image
  await updatePokemon();
}


//helper to update the image when index changes
async function updatePokemon() {
  if (pokeDex) {
    let url = pokeDex[currentIndex].image.hires;
    pokeImg = await loadImage(url);
  }
}

async function setup() {
  createCanvas(400, 400);
  await loadStuff();
  imageMode(CENTER);
}

function draw() {
  background(220);
  
  if (pokeDex && pokeImg) {
    textSize(32);
    textAlign(CENTER);
    fill(0);
    text(pokeDex[currentIndex].name.english,200,40);
    
    image(pokeImg, 200, 200, 250, 250);
 
    textSize(16);
    text("Click mouse to see the next pokemon", 200, 380);
  }
}


async function mousePressed() {
  currentIndex = (currentIndex + 1) % pokeDex.length;
  

  pokeImg = null; 
  await updatePokemon();
}
```
