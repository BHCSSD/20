# Part 1: Boolean Logic & Conditional Grammar

In p5, you used `if` statements with parentheses and curly braces:

```javascript
if (score >= 10 && hasKey) {
  // do something
}

```

In Python, the logic is identical, but the writing style (syntax) is much cleaner. 
- Python throws away parentheses and curly braces.
- Instead, it uses a **colon (`:`)** and **strict indentation (4 spaces)** to group code.

```python
if score >= 10 and hasKey:
  # do something
  print("who?")


```


## 1. Comparison Operators

Most of these look exactly like JavaScript. However, Python programmers use `==` for comparisons (Python doesn't have a `===` operator because its type system handles comparisons safely).

| Operator | Meaning | Example | Result |
| --- | --- | --- | --- |
| `==` | Equal to | `5 == 5` | `True` |
| `!=` | Not equal to | `5 != 3` | `True` |
| `>` | Greater than | `4 > 7` | `False` |
| `<` | Less than | `2 < 8` | `True` |
| `>=` | Greater than or equal to | `5 >= 5` | `True` |

> [!NOTE]
> In Python, Booleans must be capitalized: `True` and `False`. Writing `true` or `false` will cause an error
> 

## 2. Logical Operators

Your JavaScript brain will try to type `&&` and `||`. 
- Python replaces these with plain English words: `and`, `or`, and `not`.

| JS Syntax | Python Syntax | Meaning |
| --- | --- | --- |
| `&&` | `and` | **Both** conditions must be true |
| "Mark down dont like the pipes" | `or` | **At least one** condition must be true |
| `!` | `not` | Flips the boolean (`not True` becomes `False`) |

## 3. `if`, `elif`, and `else` Syntax

Instead of `else if`, Python squishes the words together into `elif`.

```python
score = 85

if score >= 90:
    print("Grade: A")
elif score >= 80:
    print("Grade: B")  #this runs because 85 >= 80
else:
    print("Grade: C or below")

```

# Part 2: String Formatting & Conditional Grammar

In p5, you combined text and variables using the `+` operator or backtick template literals:

```javascript
text(`Score: ${score}`, 10, 10);

```

In Python, we use **f-strings** (formatted string literals).
- They work exactly like JavaScript's template literals, but you put a lowercase `f` right before your opening quotation mark.

## 1. How to use f-strings

* Put an `f` before the first quote.
* Wrap your variables or math inside curly braces `{}`.

```python
player_name = "Link"
hearts = 3

# Injecting variables cleanly
message = f"{player_name} has {hearts} hearts left"
print(message)  # Output: Link has 3 hearts lef

# Doing math inside an f-string
print(f"Next level requires {hearts * 5} XP.")

```

## 2. Dynamic Grammar, The Inventory Checker

When building games, hardcoded text can look robotic `"1 lives remaining"`. 
- We combine f-strings with `if/else` checks to dynamically fix singular vs. plural grammar.

```python
item_count = 1
item_name = "potion"

if item_count == 1:
    # Singular output
    print(f"You have {item_count} {item_name}.")
else:
    # Plural output (adds an 's')
    print(f"You have {item_count} {item_name}s.")

```

What happens if you have custom plurals, like "wolf" to "wolves"?

```python
wolves_count = 3

if wolves_count == 1:
    print(f"There is {wolves_count} wolf in the area.")
else:
    print(f"There are {wolves_count} wolves in the area.")

```

# Part 3: Condition-Based Loops `while`

A `while` loop runs as long as a specific condition remains **True**. 
- It behaves identically to a JS `while` loop, but follows Python's layout rules.

```python
energy = 3

while energy > 0:
    print("Still running...")
    energy -= 1  # Subtracts 1 from energy each loop

print("Out of energy")

```

> [!WARNING]
> 
> 1. **No `++` or `--`:**
>    - Python does not support `energy--` or `i++`. You must write `energy -= 1` or `i += 1`.
> 3. **Infinite Loops:**
>    - If you forget to change the variable inside your loop (like forgetting `energy -= 1`), your condition will stay `True` forever, crashing your program
>

# Part 4: Sequence-Based Loops `for` & `range`

Before, you wrote standard counting loops like this:

```javascript
for (let i = 0; i < 5; i++) { ... }

```

Python eliminates this messy syntax. Instead of manually creating a counter variable and incrementing it, Python uses the `range()` function to automatically generate a sequence of numbers for the loop to step through.

## 1. The `range(start, stop, step)` Function

* **start**: Where the sequence begins (inclusive—defaults to 0 if omitted).
* **stop**: Where the sequence ends (**exclusive**—it stops *before* reaching this number).
* **step**: How much to add or subtract on each loop pass (defaults to 1 if omitted).

## 2. Counting Forward

If you only give `range()` one number, it treats it as the **stop** value and assumes you want to start at 0.

```python
# Loops 5 times. i starts at 0 and stops before 5.
for i in range(5):
    print(i)  # Outputs: 0, 1, 2, 3, 4

```

If you want to start at a specific number, provide a **start** and **stop**:

```python
# Starts at 1, stops before 6
for i in range(1, 6):
    print(i)  # Outputs: 1, 2, 3, 4, 5

```

## 3. Counting Backward

To count in reverse, you *must* pass all three values to `range()`, using a **negative number** for the **step**.

```python
# Start at 5, stop BEFORE 0 (so it stops at 1), step by -1
for i in range(5, 0, -1):
    print(i)  # Outputs: 5, 4, 3, 2, 1

print("Blastoff")

```

> [!WARNING]
>  If you try to write `range(5, 0)` without providing the negative `-1` step, Python will try to count *upward* from 5 to 0. Realizing that is impossible, it will silently exit, and your loop code will never run
