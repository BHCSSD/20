# Part 1: Variable Scope & Function Parameters

In p5, if you declared `let score = 0;` at the top of your file, every function could see it and change it. In Python, we practice **strict data isolation**. 
 - Functions should be like locked rooms: what happens inside stays inside.

## 1. Local Scope,  aka The Sandbox

Variables created inside a Python function are **local** to that function. The rest of your program has no idea they exist.

```python
def calculate_score():
    bonus_points = 50  # Local variable
    print(bonus_points)

calculate_score()
# print(bonus_points)  <--This will CRASH your program. NameError: name 'bonus_points' is not defined

```

## 2. Parameters & Arguments

To get data into a function, you must pass it explicitly through **parameters**.

```python
# 'base_xp' and 'multiplier' are parameters (placeholders)
def calculate_xp(base_xp, multiplier):
    return base_xp * multiplier

# 100 and 2 are arguments (the actual data passed in)
total_xp = calculate_xp(100, 2)
print(f"Total XP earned: {total_xp}")

```

## 3. Default Parameters

Python lets you provide "fallback" values for parameters. If the user forgets to provide an argument, the default value kicks in.

```python
# If no shield value is given, it defaults to 0
def take_damage(health, damage, shield=0):
    actual_damage = damage - shield
    return health - actual_damage

# Player A has no shield
print(take_damage(100, 20))  # health=100, damage=20, shield=0 -> Returns 80

# Player B has a shield
print(take_damage(100, 20, 5))  # health=100, damage=20, shield=5 -> Returns 85

```

> **Rule:** Default parameters *must* always be placed at the very end of your parameter list. Writing `def take_damage(shield=0, health, damage):` will cause a syntax error!

# Part 2: Separation of Concerns (The `main()` Pattern)

A core rule of software architecture is the **Separation of Concerns**. A computational function should never ask for user input or print raw text directly; it should just take data, calculate it, and return a result.

Instead, we isolate user interaction inside a dedicated controller function called `main()`.

## The Bad Way (Tightly Coupled)

This function is impossible to test automatically because it forces a human to type into a prompt every single time.

```python
def calculate_tax():
    price = float( input("Enter price: ") )  # BAD: User interaction inside math function
    tax = price * 0.05
    print(f"Tax is {tax}")                 # BAD: Output inside math function

```

## The Best Practice Pattern (Separated)

Keep math functions pure, and handle inputs/outputs cleanly inside `main()`.

```python
# Pure Math Function: No input(), no print()
def calculate_tax(price):
    return price * 0.05

# Pure Controller Function: Handles UI
def main():
    user_price = float( input("Enter item price: ") )
    
    # Pass the data into the calculator
    computed_tax = calculate_tax(user_price)
    
    print(f"Your calculated tax is: ${computed_tax:.2f}")

# Trigger the application
main()

```

# Part 3: Input Validation & Type Inspection

In p5, if a user typed a word instead of a number, you might get `NaN` (Not a Number). In Python, trying to convert the word `"pizza"` into an integer using `int("pizza")` will instantly crash your program with a `ValueError`.

To stop users from crashing our apps, we must validate their data *before* converting it.

## 1. Validating Strings with `.isdigit()`

The `.isdigit()` string method returns `True` if a string contains only numeric characters (`0-9`), and `False` if it contains letters, spaces, or symbols.

We combine this with a `while` loop to repeatedly trap the user until they give us a valid number:

```python
def main():
    user_input = ""
    
    # Keep looping while the input is NOT numeric

    while not user_input.isdigit():
        user_input = input("Enter your character's level (numbers only): ")
        
        if not user_input.isdigit():
            print("Invalid input! Please enter a whole number.")
            
    # Now it is 100% safe to convert to an integer
    level = int(user_input)
    print(f"Character level set to {level}")

main()

```

## 2. Type Inspection with `isinstance()`

Sometimes your functions receive data from external sources or other team members, and you need to ensure it is the right type. Python gives us `isinstance(variable, type)` to perform safety checks.

```python
def process_payment(amount):
    # Check if amount is either an int OR a float
    if not isinstance(amount, (int, float)):
        print("Error: Payment amount must be a number!")
        return False
        
    print(f"Processing payment of ${amount}")
    return True

process_payment("twenty dollars")  # Fails safely without crashing
process_payment(20.50)             # Works perfectly

```

# Part 4: The Modulo Operator (`%`)

The modulo operator (`%`) calculates the **remainder** left over after dividing one integer by another. It is a powerful tool for controlling software logic.

$$5 \div 2 = 2 \text{ with a remainder of } 1 \implies 5 \% 2 = 1$$

```python
print(5 % 2)  # Output: 1
print(4 % 2)  # Output: 0

```

## Practical Uses for Modulo

### 1. Checking for Even vs. Odd Numbers

Any number `% 2` that results in `0` is even. If it results in `1`, it is odd.

```python
def is_even(num):
    return num % 2 == 0

print(is_even(42))  # True
print(is_even(7))   # False

```

### 2. Finding Factors and Divisibility

You can check if a number is a perfect factor of another (great for mechanics like finding every 10th frame in a loop or checking leap years).

```python
# Check if a player qualifies for a power up every 5 levels
player_level = 15

if player_level % 5 == 0:
    print("Milestone reached! Unlocking milestone reward!")
else:
    print("Keep leveling up!")

```
