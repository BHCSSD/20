## Part 1: Ordered Sequences (Lists) & Math Helpers

In p5.js, you grouped items in an Array`[]`. In Python, we call this ordered collection a **List**. The syntax looks identical, but Python gives you powerful built-in functions to run operations on them instantly.

### 1. Creating and Indexing Lists

Lists maintain their order, are zero-indexed, and can change sizes dynamically.

```python
# A list of dice rolls
dice_rolls = [4, 6, 1, 6, 3]

# Reading data (Zero-indexed)
print(dice_rolls[0])  # Output: 4
print(dice_rolls[1])  # Output: 6

# Changing a specific value
dice_rolls[2] = 5
print(dice_rolls)     # Output: [4, 6, 5, 6, 3]

# Adding items to the end of a list using .append()
dice_rolls.append(2)
print(dice_rolls)     # Output: [4, 6, 5, 6, 3, 2]

```

### 2. Built-in Sequence Helpers

In JavaScript, calculating the sum or maximum value of an array requires writing a custom loop. 
- Python provides native helper functions to analyze numerical lists instantly:

```python
scores = [10, 50, 40, 90, 30]

print( max(scores) )  # Output: 90 
print( min(scores) )  # Output: 10 
print( sum(scores) )  # Output: 220 
```

## Part 2: Mutability vs. Immutability

Understanding **Mutability** (whether data can be changed in place) is crucial to keeping your applications stable and preventing hidden logic bugs.

* **Immutable (Cannot change):** Strings, Integers, Floats, Booleans.
  *  When you alter them, Python creates a brand-new copy behind the scenes.
* **Mutable (Can change directly):** Lists, Dictionaries.
  * If you pass a list into a function and change it, you modify the *original* list permanently.

### Where is the Data Leak 

Because lists are mutable, passing them into functions can cause unexpected side effects if you aren't careful.

```python
def remove_last_item(inventory_list):
    inventory_list.pop()  # Removes the final item

# Original list
my_gear = ["Sword", "Shield", "Potion"]

# Pass it to the function
remove_last_item(my_gear)

# The original list has been altered!
print(my_gear)  # Output: ['Sword', 'Shield']

```

> [!TIP]
> **Rule of Thumb:**
> If a function modifies a mutable sequence parameter directly, it alters the variable outside the function too.
> If you want to protect your original list, pass a copy of it instead: `remove_last_item( my_gear.copy() )`.

## Part 3: Mappings (Dictionaries)

In p5.js, you used JavaScript Objects to hold structural key-value properties: `let player = {hp: 100, name: "Zelda"};`.

In Python, we use **Dictionaries**.

### Dictionary Syntax Rules

1. Keys must be strings (or other immutable types) and are separated from values by a colon (`:`).
2. Key-value pairs are separated by commas.
3. Properties are looked up using bracket notation `[]` rather than dot notation.

```python
# Defining a character dictionary
enemy = {
    "name": "Goblin",
    "hp": 45,
    "defense": 5
}

# Accessing values (Must use brackets with quotes)
print(enemy["name"])  # Output: Goblin
# print(enemy.name)   <-- CRASH! AttributeError: 'dict' object has no attribute 'name'

# Updating data
enemy["hp"] = 40

# Adding a brand new key-value pair on the fly
enemy["is_boss"] = False
print(enemy) 
# Output: {'name': 'Goblin', 'hp': 40, 'defense': 5, 'is_boss': False}

```

## Part 4: Randomization & State Accumulation

Games and simulations rely on randomness and aggregating statistics across multiple tracking events.


### 2. State Accumulation Loops

Accumulation means updating a variable step-by-step inside a loop. We use this to build datasets dynamically or compute overall statistics like averages.

```python
import random

def simulate_dice_round(num_rolls):
    rolls = []
    
    # Accumulating data into a list
    for i in range(num_rolls):
        new_roll = random.randint(1, 6)
        rolls.append(new_roll) # State changes loop-by-loop
        
    return rolls

def main():
    game_data = simulate_dice_round(5)
    print(f"All rolls: {game_data}")
    
    # Calculate the average roll using our math helpers
    average = sum(game_data) / len(game_data)
    print(f"Average roll value: {average:.1f}")

main()

```

## Part 5: Test-Driven Unit Testing with `assert`

A professional deployment pipeline requires automated mathematical confirmation that core functions work before pushing updates. We do this by grouping structural validation tests inside dedicated testing functions.

### Designing a Test Suite

By extracting code verification away from standard user input loops, we can instantly guarantee that changes to code don't introduce regression bugs.

```python
# The production computational function
def calculate_loot_multiplier(streak_count):
    if streak_count >= 10:
        return 2.5
    elif streak_count >= 5:
        return 1.5
    else:
        return 1.0

# The automated isolated test suite
def test_loot_multiplier():
    print("Running calculate_loot_multiplier tests...")
    
    # Test base tier
    assert calculate_loot_multiplier(2) == 1.0
    # Test middle tier
    assert calculate_loot_multiplier(5) == 1.5
    # Test top tier boundary conditions
    assert calculate_loot_multiplier(10) == 2.5
    assert calculate_loot_multiplier(15) == 2.5
    
    print("All lookups and boundaries passed perfectly!")

# Run the test suite
test_loot_multiplier()

```

If any of those returns fail to match the expected outcome, the program will halt with an explicit `AssertionError`, identifying exactly where the logic engine broke.
