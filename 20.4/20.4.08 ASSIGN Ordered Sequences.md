> [!NOTE]
> **Writing Asserts for Lists and Dictionaries**
> When testing collections with `assert`, order matters for Lists, but **does not** matter for Dictionaries.
> * `assert [1, 2] == [1, 2]` passes, but `assert [1, 2] == [2, 1]` **fails**.
> * `assert {"a": 1, "b": 2} == {"b": 2, "a": 1}` **passes** because dictionaries look up values by keys, not sequence order.
