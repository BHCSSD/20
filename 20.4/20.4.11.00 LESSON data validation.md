## 1. The Basic Structure
A try-except block allows you to isolate code that might fail and provide a fallback plan.

[divide by 0](https://youtu.be/s_hbvRTGcUI?si=jYHHm6lxFIqfHpH8)
```python
try:
    # Code that might cause an error
    number = int(input("Enter a number: "))
    print(10 / number)
except ZeroDivisionError:
    # Runs if the user enters 0
    print("You cannot divide by zero.")
except ValueError:
    # Runs if the user enters "abc" instead of a number
    print("Invalid input. Please enter a numeric value.")
```

## 2. Checking Data Types via Type Casting
The most common way to handle data types is to "force" a conversion. If the conversion fails, the `except` block catches it.

```python
def process_age(age_input):
    try:
        # Attempt to convert input to an integer
        age = int(age_input)
    except ValueError:
        # This runs if age_input cannot be turned into an integer ( "twenty")
        print("Error: Input must be a numeric value.")
    else:
        # Runs only if the try block succeeded
        print(f"Success! Your age is {age}.")

process_age("25")    # Success
process_age("abc")   # Error
```

---

## 3. Checking for Instances (isinstance)
While type casting handles conversion, `isinstance()` verifies if a variable is already a specific type. You can use this inside a `try` block to manually `raise` an error if the type is incorrect.

```python
def calculate_area(radius):
    try:
        # Check if the input is an instance of int or float
        if not isinstance(radius, (int, float)):
            raise TypeError("Radius must be a number.")
        
        if radius < 0:
            raise ValueError("Radius cannot be negative.")
            
        return 3.14159 * (radius ** 2)
        
    except (TypeError, ValueError) as err:
        print(f"Validation Error: {err}")

calculate_area("5")   # Validation Error: Radius must be a number.
calculate_area(10)    # Success: 314.159
```

## 4. Checking Compatibility (TypeError)
Sometimes the data type is a valid variable, but the operation you are trying to perform is impossible for that specific type combo.

```python
def combine_values(a, b):
    try:
        # Fails if you mix a string and an integer
        result = a + b
    except TypeError as err:
        print(f"Data type mismatch: {err}")
    else:
        print(f"The result is: {result}")

combine_values(10, "5") # TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

## 5. Logic Flow with Else and Finally
To manage resources (like files or database connections), use `else` and `finally`.

* **else**: Runs only if no exceptions were raised in the `try` block.
* **finally**: Runs regardless of what happened. Use this for cleanup tasks.

```python
try:
    file = open("data.txt", "r")
    data = file.read()
except FileNotFoundError:
    print("The file was not found.")
else:
    print("File read successfully.")
finally:
    print("Execution finished.")
```

## 6. Summary of Type-Related Exceptions

| Exception | Occurs When... |
| :--- | :--- |
| **ValueError** | The type is correct, but the value is inappropriate (`int("hello")`). |
| **TypeError** | An operation is applied to an inappropriate type (5 + "5"`) or an instance check fails. |
| **AttributeError** | You try to call a method that doesn't exist for that data type (`[1, 2].upper()`). |

---

## 7. Best Practices
* **Use isinstance() for Logic**: Use `isinstance(x, int)` rather than `type(x) == int` because it supports inheritance.
* **[EAFP](https://en.wikipedia.org/wiki/EAFP) (Easier to Ask for Forgiveness than Permission)**: This is the Pythonic way, try the operation and catch the error. It is usually more efficient than checking types manually every time.
* **Avoid Bare Excepts**: Never use `except:` without a specific error name. It hides bugs and makes the code difficult to debug.
